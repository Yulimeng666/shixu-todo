# 个人日程待办网站 PRD

## 1. 产品概述

这是一个面向个人用户的日程待办网站，用于记录、管理和跟踪日常任务。第一版聚焦本地优先的个人使用场景，不做登录、团队协作和云同步，优先保证任务记录清晰、操作高效、数据可靠保存。

页面首屏直接进入待办工作区，不做营销页、介绍页或复杂引导页。

## 2. 产品目标

- 用户可以快速创建、编辑、完成和删除任务。
- 用户可以清楚区分今天、未来、全部、已完成四类任务。
- 用户可以通过搜索、筛选和排序快速定位任务。
- 用户可以通过全局分类体系统一管理任务分类。
- 用户刷新页面后，任务和分类数据仍然保留。

## 3. MVP 范围

### 3.1 必须实现

- 新增、编辑、删除、完成、取消完成任务
- 今天、未来、全部、已完成视图
- 任务字段：标题、备注、开始日期、截止日期、优先级、分类
- 搜索、筛选、排序
- 本地持久化
- 全局分类列表
- 左侧分类栏展示、筛选、新增、删除分类

### 3.2 暂不实现

- 登录、注册
- 云同步
- 团队协作
- 第三方日历同步
- 独立分类管理页
- 分类重命名
- AI 自动生成任务或自动排期
- 分析、遥测、埋点

## 4. 页面与展示内容

### 4.1 主界面

桌面端展示：

- 左侧导航区
- 顶部工具栏
- 主任务列表
- 任务编辑弹层

移动端展示：

- 顶部标题与新增入口
- 搜索框
- 视图切换
- 单列任务列表
- 任务编辑弹层

### 4.2 左侧导航区

需要展示：

- 应用名称
- 今天
- 未来
- 全部
- 已完成
- 分类区

分类区需要展示：

- 分类标题
- 分类标题右侧新增入口
- 全局分类列表
- 每个分类对应的任务数量
- 当前选中分类高亮态

交互规则：

- 点击分类后，直接按该分类筛选任务列表
- `默认` 分类始终存在
- `默认` 分类不可删除
- 删除其他分类后，该分类下任务统一回收到 `默认`

### 4.3 顶部工具栏

需要展示：

- 新增任务按钮
- 搜索输入框
- 优先级筛选
- 日期筛选
- 完成状态筛选
- 分类筛选
- 排序入口
- 当前筛选条件提示

分类筛选来源于全局分类列表，而不是临时从任务里聚合。

### 4.4 任务列表

每条任务展示：

- 完成状态
- 标题
- 开始日期
- 截止日期
- 优先级
- 分类
- 是否逾期
- 编辑与删除入口

### 4.5 任务表单

新增和编辑使用同一套表单。

字段包括：

- 标题，必填
- 备注，可选
- 开始日期，可选
- 截止日期，可选
- 优先级，默认中
- 分类，默认 `默认`

分类规则：

- 表单内只选择全局分类
- 不在表单内直接新增分类
- 未选择时自动归入 `默认`

## 5. 核心功能需求

### 5.1 新增任务

规则：

- 标题必填
- 标题保存前要 trim
- 空标题不能保存
- 优先级默认 `medium`
- 完成状态默认 `false`
- 未选择分类时自动设为 `默认`
- 创建成功后立即写入本地存储

今天视图下的新增规则：

- 当用户位于“今天”视图且未填写截止日期时，系统自动将截止日期设为今天
- 该规则只作用于“今天”视图
- 新任务应立即出现在今天列表中

### 5.2 编辑任务

允许修改：

- 标题
- 备注
- 开始日期
- 截止日期
- 优先级
- 分类

### 5.3 删除任务

- 删除前必须确认
- 删除后从当前列表移除
- 删除结果同步写入本地存储

### 5.4 完成与取消完成

- 完成时写入完成时间
- 取消完成时清空完成时间

### 5.5 分类体系

第一版分类能力包括：

- 预置分类：`默认`、`工作`、`生活`、`学习`、`购物`
- 左侧分类栏新增分类
- 左侧分类栏删除分类
- 左侧分类栏按分类筛选
- 工具栏按分类筛选

规则：

- 分类名称 trim 后不能为空
- 分类名称不能重复
- `默认` 分类不可删除
- 删除分类后，相关任务自动回收到 `默认`
- 历史任务无分类时，自动归入 `默认`

### 5.6 搜索

- 搜索范围包括标题和备注
- 搜索结果受当前视图和当前分类筛选影响

### 5.7 筛选

支持：

- 优先级筛选
- 日期筛选
- 完成状态筛选
- 分类筛选

### 5.8 排序

支持：

- 截止日期
- 优先级
- 创建时间

## 6. 设计要求

- 整体风格为简洁效率型工具
- 左侧分类栏要像工作区导航，而不是设置页
- 分类新增入口要靠近“分类”标题
- 分类项要便于快速扫描和点击
- 删除分类入口需要克制，不应压过主要筛选操作
- 移动端 320px 宽度下不能出现文字重叠和按钮溢出

## 7. 用户流程

### 7.1 创建任务

1. 用户点击新增任务
2. 系统打开任务表单
3. 用户填写标题、备注、开始日期、截止日期、优先级、分类
4. 若未选择分类，系统提交时自动补成 `默认`
5. 若当前为今天视图且截止日期为空，系统自动补今天日期
6. 系统创建任务并更新列表
7. 系统保存到本地数据库

### 7.2 管理分类

1. 用户在左侧分类标题旁点击新增
2. 输入分类名称并确认
3. 新分类立即出现在左侧和任务表单中
4. 点击某个分类后，任务列表立即按该分类筛选
5. 删除某个非默认分类时，系统要求确认
6. 删除后，该分类下任务统一回收到 `默认`

## 8. 验收标准

- 用户可以新增、编辑、删除、完成、取消完成任务
- 用户可以在今天、未来、全部、已完成视图之间切换
- 用户可以录入并展示开始日期
- 用户可以使用全局分类列表选择分类
- 左侧分类栏可以新增分类并立即生效
- 左侧分类栏可以删除非默认分类
- 删除分类后，相关任务自动回收到 `默认`
- 新建任务未选择分类时，自动归入 `默认`
- 今天视图中新建未填截止日期任务后，任务即时可见
- 工具栏分类筛选与左侧分类点击保持一致
- 刷新页面后，任务和分类数据仍然存在

## 10. 视觉主题补充

- 网站可以使用暖色调背景，但不能让所有区域融成同一层颜色。
- 左侧导航、顶部工具栏、当前视图信息区、任务列表区必须通过清晰边框分区。
- 暖色主题下优先使用浅暖底色加白色或半白内容面，而不是整页同色铺满。
- 任务列表容器、导航容器、筛选工具栏至少保留一层可见边框，方便快速识别区域。
- 背景氛围可以偏暖，但任务内容区必须保持足够对比度，不能影响扫描效率。
